# REST API

# Wprowadzenie do API REST'owego serwera Tower

Sewer Tower korzysta z serwera loopback i wystawiane API CRUD'owe jest z nim kompatybilne.

# Explorer

Aby zapoznać się z wszystkimi funkcjami API Tower, należy uruchomić Tower z opcją explorer. Następnie należy uruchomić otworzyć przez przeglądarkę URI /explorer

# Logowanie

Większość metod wystawionych przez API Tower, wymaga autoryzacji i autentykacji. Innymi słowy, aby wykonać metodę w Tower, trzeba najpierw się zalogować, używając metody login z modelu member, gdzie parametr credentials jest wypełniony jak poniżej:

{
  "username":"admin",
  "password":"admin"
}

Oczywiście należy wpisać odpowiednie dane użytkownika i jego hasło. W odpowiedzi na poprawne żądanie otrzymamy wiadomość podobną do poniższej:

{ 
  "id": "2VHKDb5LvW9ogWUrwFFOB5d0AKy1NxJgAzwBFu895xBIaHARKfiFCrBl2fLTWgtV", 
  "ttl": 1209600, 
  "created": "2019-08-03T10:20:27.899Z", 
  "userId": "5d3dd6907ecf764681448931" 
}

Pole id z odpowiedzi to token, którym możemy się przedstawić wywołując API Tower.

# Wywoływanie API

API Tower wywołuje się jak zwykłe API REST'owe, z tą różnicą, że większość metod jest zabezpieczona. Są dwie metody aby podać token, który jest potrzebny, aby się poprawnie przedstawić w Tower:

# Token w nagłówku

Jedną z metod podania tokena utentykującego użytkownika w Tower, jest przekazanie tokena w nagłówku HTTP. Co oznacza, że przy każdym wywołaniu API, należy przekazać nagłówek Authorization z wartością tokenu otrzymanego przy logowaniu, np.

Authorization: oirFTMeE2rY6tXuSeQ7bNtStxdt82MatnshpjXYgWGBv0gxHDaYo6Vh74vjImdDf

# Token w URI

Następną metodą podania tokena przy wywołaniu API Tower, jest przekazanie go w parametrze URI, innymi słowy, aby wywołać funkcję z tokenem, trzeba oprócz zwykłego URI oraz standardowych parametrów, przekazać również wartość access_token, np.

http://tower/api/configuration?access_token=oirFTMeE2rY6tXuSeQ7bNtStxdt82MatnshpjXYgWGBv0gxHDaYo6Vh74vjImdDf
