# Konfiguracja

# Pliki konfiguracyjne

.
├─ config.json

# config.json

Plik config.json odpowiada za główną konfigurację Tower. W nim znajdują się informacje jaki host/port służy do wystawienia serwera oraz czy połączenie jest szyfrowane

#przykładowa konfiguracja
{
	"host": "0.0.0.0",
	"port": 5000,
	"logLevel": "error",
	"explorer": true,
	"privateKey": "./certs/privatekey.pem",
	"certificate": "./certs/certificate.pem"
}

# host

Adres, na którym ma zostać wystawiony serwer.

"host": "0.0.0.0"

# port

Port, na którym ma zostać wystawiony serwer.

"port": 3000

# explorer (opcjonalny)

Ustawienie tej zmiennej na true powoduje, że Tower wystawia Loopback API Explorer, który pozwala na swobodne przeglądanie API REST'owego pod nowym URI wystawionym przez Tower /explorer

"explorer": true

# logLevel (opcjonalny)

Poziom logowania serwera Tower. Domyślna wartość to 'error'

"logLevel": "error",

Dozwolone wartości:

  • error
  • warn
  • info
  • verbose
  • debug
  • silly

# privateKey (opcjonalny)

Ścieżka do pliku z kluczem prywatnym w formacie PEM.

"privateKey": "./certs/privatekey.pem"

# certificate (opcjonalny)

Ścieżka do pliku z certyfikatem w formacie PEM.

"certificate": "./certs/privatekey.pem"

# database-config.json

Plik database-config.json odpowiada za konfigurację bazy danych. Aktualnie Tower wspiera tylko bazy danych MongoDB

#przykładowa konfiguracja
{
	"mongoDB": {
		"hostname": "mongodbHost",
		"port": 27017,
		"database": "tower_prod",
		"password": "pass",
		"user": "user"
	}
}

# hostname

Host bazy danych MongoDB, do której ma łączyć się Tower.

"hostname": "mongodbHost"

# port

Post bazy danych MongoDB, do której ma łączyć się Tower.

"port": 27017

# database

Nazwa bazy danych, z której ma korzystać Tower.

"database": "tower_prod"

# password

Hasło użytkownika, którym Tower będzie się łączył do bazy danych

"password": "pass"

# user

Nazwa użytkownika, którym Tower będzie się łączył do bazy danych

"user": "dbUser"

# providers.json

Plika zawiera konfigurację klienta LDAP, Więcej informacji na temat tej konfiguracji można znaleźć na stronie passport.js, z implementacji której korzysta Tower.

{
	"ldap": {
		"server":{
			"url": "ldap://ldapHost:389",
			"bindDN": "cn=admin,dc=example,dc=org",
			"bindCredentials": "admin",
			"searchBase": "dc=example,dc=org",
			"searchFilter": "(cn={{username}})"
		}
	}
}

# url

Url serwera LDAP

"url": "ldap://ldapHost:389"

# bindDN

DN użytkownika mającego uprawnienia do wyszukiwania i autentykowania innych użytkowników na serwerze LDAP.

"bindDN": "cn=admin,dc=example,dc=org"

# bindCredentials

Hasło użytkownika bindDN.

"bindCredentials": "admin"

# searchBase

Baza wyszukiwania w LDAP

"searchBase": "dc=example,dc=org"

# searchFilter

Filtr szukania użytkownika. Aby wprowadzić nazwę użytkownika do wyszukiwania, należy użyć wartości username jak poniżej

"searchFilter": "(cn={{username}})"
