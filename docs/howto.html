# Jak używać Tower?

# Założenia

W Tower konfiguracja przechowywana jest stałej strukturze Projekt -> Środowisko -> Komponent -> Wersja. Czyli, w projekcie możemy mieć wiele środowisk, które mogą zawierać wiele konfiguracji danego komponentu w konkretnej wersji.

Każda konfiguracja też jest wersjonowana, co oznacza, że mamy pełny wgląd w historię konfiguracji z podziałem na daty jej obowiązywania.

# Pierwsze uruchomienie

# Pierwsze kroki

Po uruchomieniu przeglądarki internetowej i wejściu na adres URL wystawiony przez serwer Tower (zgodnie z konfiguracją host i port), widzimy stronę logowania. Do pierwszego logowania używamy użytkownika admin z hasłem admin.

Od razu po pierwszym logowaniu użytkownik jest proszony o zmianę hasła.

# Konfiguracja

Pierwszą rzeczą, którą należy zrobić po poprawnym zalogowaniu jako użytkownik admin, jest dodanie nowego środowiska. Aby to zrobić, przejść do zakładki settings->environments. Tam w polu Environment wpisujemy nazwę środowiska, które chcemy stworzyć i klikamy przycisk +.

# Tworzenie bazowej konfiguracji

Aby stworzyć nową konfigurację, pierwszą rzeczą, którą należy zrobić, jest dodanie "konfiguracji bazowej", czyli informacji jakie zmienne zawiera konfiguracja, jakiego są one typu i jakie mają domyślne wartości. Aby stworzyć taką konfigurację, należy wywołać API REST'owe serwera Tower spod URI /configurations/addComponent, pamiętając o tym, że tylko zalogowany użytkownik może wykonywać zapytania do API. Dane, które są przyjmowane w body zapytania, powinny zawierać informacje o zmiennych komponentu, którego konfigurację chcemy przechowywać, np:

[
  {
    "name":"String Variable",
    "type":"string",
    "value":"initial value"
 },
 {
    "name":"Number Variable",
    "type":"number",
    "value": 1234
 },
 {
    "name":"Boolean Variable",
    "type":"boolean",
    "value": false
 },
 {
    "name":"Password variable",
    "type":"password",
    "value": "My_SeCrEt"
 }
]

W tym wypadku przekazujemy 4 zmienne. Każda z tych zmiennych ma swoją nazwę name, typ type oraz wartość value. Wartość name pozawala nam nazwać naszą zmienną. Może być to dowolny ciąg znaków, lecz nie może się od powtarzać w obrębie jednej konfiguracji.
Zmienna value przedstawia domyślną wartość dla naszej zmiennej.
Wartość type pozwala nam określić jakiego typu jest wartość. Dozwolone typy to string, number, boolean i password. Każdy z tych typów ma zachowuje się inaczej:

# string

String to zwykły ciąg znaków, jest to domyślny typ dla wszystkich wartości w Tower.

# number

Number jest dowolną wartością numeryczną.

# boolean

Boolean przedstawia zmienną logiczną przyjmującą wartości true lub false

# password

Password zachowuje się podobnie do wartości string, ale w bazie danych jest przechowywana w postaci zaszyfrowanej.
